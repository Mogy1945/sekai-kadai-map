# 世界問題課題マップ

各国が直面する社会課題を、**実データに基づく深刻度スコア**で可視化したインタラクティブマップ。

収録: 🇯🇵日本17 / 🇰🇷韓国14 / 🇨🇳中国11 / 🇹🇼台湾12 / 🇰🇵北朝鮮8 の大課題62(サブ課題311)
＋ 🌐**国際共通課題9**(台湾海峡・南シナ海と九段線・北朝鮮核ミサイル・東シナ海尖閣・越境大気汚染・半導体サプライチェーン・東アジア同時少子化・気候変動・エネルギー依存)。

- **2Dのちゃんとした世界地図**上で、国をタップすると課題ネットワークが展開。円の大きさ＝深刻度
- 全ネットワークは**地理方向を保った1つの連続空間**にあり、パンするだけで隣国のマップへスライドできる。画面外の隣国へは方向チップから飛行移動
- **国際共通課題は国と国の間の海上に浮かび**、関与国の課題とオレンジの破線で接続(計77本)。パネルには国別の影響と各国課題へのジャンプ付き
- 深刻度スコア = 影響人数(40%) × 経済インパクト(35%) × 緊急度(25%) — 国横断で同一基準の機械算出
- 全数値に**出典リンク付き**。政治的に立場が分かれる論点は両論併記。北朝鮮の数値は推計主体を明記
- ラベルはGoogleマップ式に自動間引き(スコア優先)。ズームインで全表示
- オフライン動作の単一HTML。スマホ対応(ピンチズーム/ボトムシート)。静止時CPU約1%

## 遊び方

`index.html` をブラウザで開くだけ。

- 地球 → 国選択ボタン or 地図上のクラスタをタップ
- ノードをタップ → 詳細パネル(スコア内訳・統計・出典・関連課題)
- 背景ドラッグでパン(隣国までそのまま行ける) / ホイール・ピンチでズーム / ノードはドラッグ可
- 「一覧」= スコア順テーブル(国別+国際) / 「大きさの根拠」= スコア算出方法

## データについて

課題の選定・数値は制作者の意見ではなく、国ごとにマルチエージェント調査で構築:

1. **3視点タクソノミー**: 政府統計・白書 / OECD等の国際比較 / シンクタンク・世論調査 から独立に抽出し統合
2. **課題別調査**: 課題ごとに最新の一次資料から統計・出典を収集
3. **敵対的ファクトチェック**: 別エージェントが全数値・出典URLを一次資料と突合し修正(5カ国+国際で計約700件の修正を反映・低信頼判定0件)
4. **抜け漏れ批評**: 補完調査(日本のみ実施済み)

国際共通課題は、確定した62課題のIDリストを渡した別ワークフローが「複数国に実質的に跨がるもの」だけを抽出・検証。
中国は公式統計と独立推計の乖離を両論併記、北朝鮮は全数値に推計主体を明記。

## 構成

- `index.html` — 本体(単一HTML・ビルド成果物)
- `src/graph.js` — スコア計算+力学シミュレーション(UI非依存・node直require可)
- `src/data.js` — 課題データセット(出典付き・gen-data.mjsで生成)
- `src/mapdata.js` — 世界地図(Natural Earth由来・等距円筒図法)
- `src/app.js` / `style.css` / `template.html` — UI層
- `build.mjs` — ビルド(レイアウトの事前計算ベイク込み)
- `sim/e2e.cjs` — E2E(データ整合性+物理+jsdom UIフロー+決定性)
- `tools/` — 地図変換 / データ生成 / スクリーンショット / 実ブラウザタップ検証 / 性能計測

## 開発

```bash
node build.mjs         # ビルド
node sim/e2e.cjs       # テスト
node tools/uitest.mjs  # 実ブラウザ操作検証(要playwright)
node tools/shots.mjs   # スクリーンショット
node tools/perf.mjs    # 性能計測
# 国の追加: tools/build-map.mjs のREGION_IDSにISO3を追加 → 調査ワークフロー → tools/gen-data.mjs
```
